\begin{abstract}
面向金融风控、反欺诈、反洗钱与交易监测等场景，异常检测系统需要在大规模数据中识别少量高风险样本，并同时满足吞吐率和时延要求。这类任务通常具有异常比例低、数据形态多样和业务时效性强等特点，离线可用的检测方法若缺少系统化执行支撑，难以直接用于近实时链路。随着交易规模增长，系统瓶颈逐渐从单一模型精度转向异常评分复杂、候选召回访问开销、CPU 与 GPU 数据交换以及多 GPU 扩展效率等端到端问题。仅对局部算子或检索模块加速，难以消除候选召回、异常评分和并行扩展之间的链路开销。

针对上述问题，本文设计并实现 PyTOD--FlashANNS 协同异常检测系统 FlashTOD。PyTOD 是基于张量操作的异常评分框架，FlashANNS 是面向大规模外存近似最近邻检索的候选召回系统。FlashTOD 先通过 FlashANNS 在大规模样本中生成候选集合，再由 PyTOD 在候选范围内完成张量化异常评分，并输出异常分数和前 $k$ 个结果（Top-$k$）风险对象。通过候选预算调节评分范围，FlashTOD 在检测效果和系统开销之间形成可控折中。该系统的重点是组织两类已有组件之间的数据流、接口和执行顺序，而不是提出新的异常检测理论模型。

在工程实现上，单 GPU 路径围绕减少 CPU--GPU 往返、候选重组和阶段同步展开。系统结合 FlashANNS 异步 I/O--计算流水线、页锁定内存（pinned memory）、RAPIDS GPU 数据处理生态和 FAISS 的 GPU 检索实现（FAISS-GPU），尽量保持查询特征、候选结果和评分中间张量在 GPU 侧连续传递。多 GPU 路径则以降低跨卡归并和共享 I/O 竞争为目标，采用索引复制式 replicated 架构组织数据并行；在需要跨卡归并时，借助 NVIDIA 集合通信库（NCCL）下沉设备间通信，并结合拓扑感知的 I/O 通道（I/O lane）绑定与准入控制缓解共享路径冲突。这些组件均作为工程支撑被引入和适配，作用是压缩路径开销，而不是替代异常检测模型本身。

实验部分使用约 1M 规模真实或半真实金融数据进行有效性验证，并将 10M、50M 与 100M 高保真合成数据用于吞吐、延迟和扩展性压力测试。在本文设定的数据规模、硬件平台和候选预算条件下，完整融合方案的精确率—召回率曲线下面积（PR-AUC）为 0.906，接近 PyTOD 全量评分的 0.932；每秒查询数（QPS）由全量评分的 2700 提升至 4280，约提升 58.5\%。单卡完整优化方案将 QPS 从 3180 提升至 4550，约提升 43.1\%，并将第 99 百分位延迟（p99 延迟）从 67.0 ms 降至 45.5 ms，约降低 32.1\%。在 replicated 路径下，1/2/4 GPU 的 QPS 分别为 4550、7780 和 13620，4 GPU 约为 1 GPU 的 2.99 倍。上述结果表明，FlashTOD 在效果保持、单卡路径压缩和多卡吞吐扩展方面具有可验证的工程收益；相关结论仍受数据规模、硬件平台和候选预算约束。总体而言，本文为高吞吐、近实时金融异常检测系统提供了一条系统实现路径。
\end{abstract}

\begin{abstract*}
In financial risk control, fraud detection, anti-money laundering, and transaction monitoring, outlier detection systems need to identify a small number of high-risk samples from large-scale data while meeting throughput and latency requirements. Such tasks usually involve low anomaly ratios, heterogeneous data forms, and strict timeliness requirements, so an offline detection method cannot be directly used in a near-real-time pipeline without system-level execution support. As transaction volume grows, the bottleneck shifts from the accuracy of a single model to end-to-end costs, including expensive anomaly scoring, candidate retrieval overhead, CPU--GPU data movement, and limited multi-GPU scalability. Accelerating only an individual operator or retrieval module is therefore insufficient to remove the overhead across candidate retrieval, anomaly scoring, and parallel scaling.

To address these issues, this thesis designs and implements \emph{FlashTOD}, a collaborative outlier detection system built with PyTOD and FlashANNS. PyTOD is an anomaly scoring framework based on tensor operations, and FlashANNS is a candidate retrieval system for out-of-core approximate nearest neighbor search. FlashTOD first uses FlashANNS to generate a candidate set from large-scale samples, then applies PyTOD to perform tensorized anomaly scoring within the candidate set, and finally outputs anomaly scores and Top-$k$ risky objects. By controlling the candidate budget, FlashTOD provides a tunable trade-off between detection effectiveness and system cost. The focus is the organization of data flow, interfaces, and execution order between two existing components, rather than a new theoretical model for outlier detection.

In the engineering implementation, the single-GPU path is organized around reducing CPU--GPU round trips, candidate repacking, and stage synchronization. The system combines the asynchronous I/O--compute pipeline of FlashANNS, pinned memory, the RAPIDS GPU data-processing ecosystem, and FAISS-GPU, the GPU retrieval implementation of FAISS, so that query features, candidate results, and intermediate scoring tensors can remain on the GPU as much as possible. The multi-GPU path aims to reduce cross-GPU merging overhead and shared I/O contention. It uses an index-replicated architecture for data parallelism, employs NCCL for inter-device communication when cross-GPU merging is needed, and applies topology-aware I/O lane binding with admission control to mitigate shared-path conflicts. These components are introduced and adapted as engineering support for reducing path overhead, rather than as replacements for the anomaly detection model.

Experiments use real or semi-real financial data at around 1M scale for effectiveness evaluation, and high-fidelity synthetic data at 10M, 50M, and 100M scales for throughput, latency, and scalability stress tests. Under the evaluated data scales, hardware platform, and candidate budgets, the full fusion scheme achieves a PR-AUC of 0.906, close to the 0.932 of the PyTOD full-scoring baseline. Its throughput increases from 2700 QPS to 4280 QPS, a 58.5\% improvement. The full single-GPU optimization increases QPS from 3180 to 4550, a 43.1\% improvement, and reduces p99 latency from 67.0 ms to 45.5 ms, a 32.1\% reduction. Along the replicated path, the QPS values on 1, 2, and 4 GPUs are 4550, 7780, and 13620, respectively, with 4 GPUs reaching about 2.99$\times$ the 1-GPU throughput. These results show verifiable engineering gains in effectiveness preservation, single-GPU path compression, and multi-GPU throughput scaling, while the conclusions remain bounded by the evaluated data scales, hardware platform, and candidate budgets. Overall, FlashTOD provides a system implementation path for high-throughput and near-real-time financial outlier detection.
\end{abstract*}
